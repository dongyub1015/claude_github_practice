# 설계: Phase 8 — 제한 시간

## 목표 요약

60초 카운트다운 타이머를 HUD에 표시하고, 시간이 0이 되면 사망 처리(잔기 소모)한다. 사망 시 타이머가 60초로 초기화된다.

---

## 상수

| 상수 | 값 | 설명 |
|------|-----|------|
| `TIME_LIMIT` | 60 | 제한 시간(초) |

---

## 타이머 상태

```ts
const timerRef = useRef(TIME_LIMIT * 60)  // 프레임 단위 (60fps 기준 3600)
```

---

## 타이머 동작 규칙

| 상황 | 타이머 |
|------|--------|
| 정상 게임 중 | 매 프레임 -1 |
| 리스폰 대기 중 (`respawnTimer > 0`) | 정지 (카운트 안 함) |
| 사망 후 리스폰 시 | **60초로 초기화** |
| 클리어 / 게임 오버 | 정지 |

---

## 시간 초과 처리

타이머가 0에 도달하면 플레이어-풍선 충돌과 동일한 사망 처리를 수행한다.
같은 프레임에 타이머 소멸과 충돌이 동시에 발생할 경우 **타이머 소멸을 우선** 처리한다.

```
timerRef.current -= 1
timerRef === 0
  └─ 사망 처리
       ├─ lives > 0 → 리스폰 + timer 초기화(3600)
       └─ lives === 0 → 게임 오버
else
  └─ 플레이어-풍선 충돌 감지 → 사망 처리 (기존)
```

---

## HUD 업데이트

`drawHUD(ctx, lives, timer)` 에 timer 파라미터 추가.

```
┌─────────────────────────────────────────┐
│ LIFE  ● ● ●              TIME  60       │
└─────────────────────────────────────────┘
```

- 표시값: `Math.ceil(timer / 60)` → 1~60초 (올림)
- 10초 이하: 빨간색(`#ff4400`)으로 강조
- 2자리 0-패딩: `'05'`, `'60'`

---

## 게임 루프 구조 변경

```
loop()
  └─ update (cleared / gameOver 아닐 때)
       ├─ respawnTimer 카운트다운
       ├─ 플레이어 이동 / 풍선 물리 / 와이어 (항상)
       ├─ 와이어-풍선 충돌 & 분열 (항상)
       ├─ 클리어 판정 (항상)
       └─ respawnTimer === 0 일 때만:
            ├─ timer -= 1
            ├─ timer === 0 → 사망 처리 + timer 초기화  (신규)
            └─ else → 플레이어-풍선 충돌 → 사망 처리
  └─ render
       └─ drawHUD(ctx, lives, timer)  ← timer 인자 추가
```
