# 설계: Phase 1 — 메인 화면

## 목표 요약

브라우저 실행 시 PANG 메인 화면이 출력되고, 키보드로 메뉴를 탐색하여 게임을 시작하거나 종료할 수 있다.

---

## 화면 레이아웃

```
┌──────────────────────────────────────┐
│                                      │
│              P  A  N  G              │  ← 타이틀 (대형 텍스트)
│                                      │
│          HI-SCORE  000000            │  ← 최고 점수
│                                      │
│                                      │
│            ▶ 1 PLAYER                │  ← 선택된 항목 (커서 ▶ 표시)
│              게임 종료               │
│                                      │
│                                      │
└──────────────────────────────────────┘
```

- 배경: 단색 검정 (`#000000`)
- 전체 화면 중앙 정렬
- 게임 화면 고정 크기: **480 × 640px** (세로형)

---

## 구성 요소

### 1. 타이틀 텍스트
- 문자열: `PANG`
- 위치: 화면 상단 중앙 (세로 기준 약 25% 지점)
- 스타일: 레트로 픽셀 계열 폰트, 흰색 대형 텍스트

### 2. HI-SCORE
- 문자열: `HI-SCORE  000000`
- 위치: 타이틀 아래
- 세션 내 최고 점수를 6자리 0-패딩으로 표시
- 초기값: `000000`

### 3. 메뉴 항목
| 인덱스 | 항목 | 동작 |
|--------|------|------|
| 0 | 1 PLAYER | 게임 화면으로 전환 |
| 1 | 게임 종료 | 종료 안내 문구 표시 |

- 선택된 항목: 좌측에 `▶` 커서 표시, 노란색(`#FFD700`)
- 미선택 항목: 커서 없음, 흰색(`#FFFFFF`)

### 4. 종료 안내 문구
- 게임 종료 선택 시 메뉴 아래에 노출
- 내용: `브라우저 탭을 닫아 종료하세요.`
- 별도 화면 전환 없이 현재 화면에서 인라인으로 표시

## 컴포넌트 구조

```
App.tsx
└── MainScreen.tsx          ← Phase 1 진입점
    ├── Title.tsx            ← "PANG" 타이틀 텍스트
    ├── HiScore.tsx          ← HI-SCORE 표시
    └── MenuList.tsx         ← 메뉴 항목 목록
        └── MenuItem.tsx     ← 개별 메뉴 항목 (커서 포함)
```

`App.tsx`는 현재 화면 상태(`'main' | 'game'`)를 관리하고, Phase 1에서는 항상 `'main'`을 렌더링한다.

## 입력 처리

### 키보드

`useEffect` 안에서 `keydown` 이벤트를 등록하고, 컴포넌트 언마운트 시 제거한다.

| 키 | 동작 |
|----|------|
| `ArrowUp` | 이전 메뉴 항목으로 이동 (순환) |
| `ArrowDown` | 다음 메뉴 항목으로 이동 (순환) |
| `Enter` | 현재 선택 항목 실행 |

순환 처리: 첫 항목에서 ↑ 누르면 마지막 항목으로, 마지막 항목에서 ↓ 누르면 첫 항목으로 이동.

### 마우스

`MenuItem` 컴포넌트에 `onMouseEnter`, `onClick` 이벤트를 부여한다.

| 이벤트 | 동작 |
|--------|------|
| `onMouseEnter` | 해당 항목으로 선택 이동 (커서 `▶` 이동), 종료 안내 문구 숨김 |
| `onClick` | 해당 항목 즉시 실행 |

키보드와 마우스 조작은 동일한 `execute(index)` 함수를 공유한다.

## 화면 전환

```
메인 화면
  ├── 1 PLAYER 선택 → Enter
  │     └── App.tsx의 screen 상태를 'game'으로 변경 (Phase 2에서 구현)
  │           ※ Phase 1에서는 전환 없이 콘솔 로그로 대체
  └── 게임 종료 선택 → Enter
        └── showExitMessage = true → 안내 문구 표시
```

## 스타일 방향

- CSS 파일(`MainScreen.css`)로 분리 관리
- 배경: 검정, 텍스트: 흰색 기본 / 선택항목 노란색
- 폰트: `'Press Start 2P'` (Google Fonts, 픽셀 레트로 계열) 사용 예정
  - 로드 실패 시 fallback: `monospace`
- 레이아웃: Flexbox 세로 정렬, 전체 중앙 배치

## 검토 요청 사항

아래 항목에 대해 피드백 부탁드립니다.

1. **화면 크기** — 480×640px (세로형)이 적절한지, 가로형으로 바꿀지 여부
2. **게임 종료 동작** — 인라인 안내 문구 방식이 괜찮은지, 다른 방식 선호 여부
3. **폰트** — 픽셀 레트로 폰트 방향이 맞는지 여부
